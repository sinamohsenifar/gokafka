# Activity Log

Append-only log of wiki ingests and notable project events. Newest first.

## 2026-07-01
- **v0.26.22–v0.26.27 shipped — DATA-LOSS / MESSAGE-INTEGRITY AUDIT complete (6 confirmed findings, one PR+release each).** An adversarial find→verify Workflow over the produce/consume/rebalance/share flows surfaced 16 candidates → 6 real integrity bugs; a second design+verify Workflow adversarially reviewed the fixes and caught draft regressions (slice-index leader lookup, per-broker vs per-partition ack granularity, offset- vs position-based fetch truncation). Fixes: **CRC** record-batch CRC-32C validation on consume (v0.26.22); **idempotent seq per-record** not per-batch + **fatal OUT_OF_ORDER/epoch** (was retriable + reset-PID-and-resend → dup) (v0.26.23); **OffsetFetch completeness** — never resume an assigned partition at offset 0 on a transient/omitted response (v0.26.24); **freeze partition across produce retries** (RoundRobin re-partition on retry) (v0.26.25); **Poll delivered-position cursor** — multi-broker parallel fetch + `out[:maxPoll]` truncation was dropping already-cursor-bumped records = silent loss (v0.26.26); **per-partition produce retry** — partial multi-broker failure re-sent the whole batch → dup of committed partitions when idempotence off (v0.26.27, validated on a real 3-broker cluster). Each has a regression test (incl. new `kfake` fault-injection knobs `FailNextOffsetFetch`/`FailNextProduce`/`FailNextProducePartition`). All CI green (Kafka 3.9.2/4.3.0 matrix + Redpanda).
- **v0.26.21 shipped** — producer send-path perf: removed a dead `protoByKey` allocation (built per produce, never read). BenchmarkProduceSync1000 (via kfake): 92→80 allocs, 2.74MB→2.34MB, ~20% faster. Behaviour-identical.
- **v0.26.18–v0.26.20 shipped — hardening → performance → docs arc.** (1) Fuzzed the wire decoders (12 `go test -fuzz` targets) — found + fixed a real DoS/panic (unbounded/negative `make` from wire counts in `assignment.go`/`group_assign.go`); de-flaked `TestIntegrationAdminACL`. (2) Rewrote the record-batch decoder to a single per-batch backing buffer: **2014→4 allocs, 116µs→57µs, ~2× consume throughput** on the decode path. (3) Added `Admin.ShareGroupLag` + [`docs/SHARE_GROUPS.md`](../docs/SHARE_GROUPS.md) + `examples/sharegroupadmin`. See [[Audit: KIP-932 implementation gaps]].
