# Session 2026-06-30 — competitor parity + Redpanda

A long working session that took GoKafka from "feature-complete vs Kafka" to "matches/exceeds all four alternatives and is Redpanda CI-verified."

## Phase 1 — KIP-890 TV2 (continued)
Landed the [[features/exactly-once-tv2|TV2]] foundation + gated produce path. The first attempt was caught by integration tests (it would have broken 4.x transactions); the fix required Produce v12 and a verified hybrid. See [[decisions/adr-tv2-incremental]].

## Phase 2 — competitor analysis & parity
Researched [[competitors/franz-go|franz-go]], [[competitors/sarama|sarama]], [[competitors/kafka-go|kafka-go]], [[competitors/confluent-kafka-go|confluent-kafka-go]] in parallel, then closed every realistic gap (9 features) — see [[competitors/parity-matrix]]. Highlights: [[packages/kfake-mock-broker|kfake mock broker]] and [[features/csfle|CSFLE]].

## Phase 3 — GitHub repo polish
Set the repo description, pkg.go.dev homepage, and 20 topics (the page was bare). Added partition reassignments, header-based SR framing, `SchemaByGUID`.

## Phase 4 — Redpanda support
Tested empirically against a real Redpanda broker. Core worked immediately; testing surfaced **three real protocol bugs** ([[compatibility/broker-quirks]]) latent on Kafka. Added a [[compatibility/redpanda|Redpanda CI lane]] and made the suite broker-portable.

> [!quote] Key lesson
> Testing against a second Kafka-compatible broker exposes latent flexible-decode bugs that null-valued Kafka responses hide. See [[protocol/flexible-encoding]].

## Outcome
Shipped through **v0.26.7**. Full suite `-race` green against both Kafka (3.9–4.3) and Redpanda. See [[history/releases]].
